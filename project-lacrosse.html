Lacrosse Expected Goals Model

This data science project combined my passion for lacrosse with machine learning to create an expected goals (xG) model. I undertook this project to learn data analysis methods in Python, as I had only learned R in my coursework. Using classification techniques, I developed a model that quantifies the probability that a specific shot will result in a goal based on various factors such as shot location, angle, and game situation.

A key component of this project was using computer vision packages in Python, specifically OpenCV, to track player and ball information from lacrosse game film. I used homography to map the location of the shooter on the field, which allowed me to accurately determine shot positions and angles.

The final model achieved an ROC_AUC of 0.8333, which demonstrates a relatively strong ability to correctly predict goals. This result shows the model's effectiveness in distinguishing between shots that will result in goals and those that won't.

Next steps to improve this project include using Bootstrapping and Cross Validation to better ensure there is no overfitting in the model. I would also use hyperparameter tuning to help create a more precise model. Additionally, I would create a more robust dataset that includes more shots from a wider variety of games and teams. To accomplish this, I would train a computer vision model that recognizes and tracks players live in film and directly scrapes the data. Lastly, I would also experiment with different variables and try different classification models such as XGBoost and Random Forest. In using these different models I would be able to include the partial data of the shot clock because XGBoost can deal with NA values.
